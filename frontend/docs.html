02. Problem Statement

Modern software development teams suffer from disjointed operational silos and lack of integrated DevSecOps pipelines.

The Core Challenges

  • Isolated Systems: CI/CD platforms, security scanners, database consoles, and infrastructure monitoring dashboards exist in disconnected networks, requiring multiple VPNs and credentials.
  • Spreadsheet-Based Tracking: Project metrics, pipeline statuses, and critical vulnerability counts are compiled manually, causing human delays and stale operational views.
  • Security Governance Gaps: Developers often deploy infrastructure containing vulnerability CVEs without compliance guardrails.

03. Solution Architecture

The system uses a highly secure, modular architecture running on AWS EC2 behind an Nginx Reverse Proxy with secure Docker container isolation.

Architecture Breakdown

  • Gateway Routing: An Nginx container routes HTTP requests on port 80 to the frontend service and proxies `/api` endpoints and WebSocket traffic (`/socket.io`) to the backend API.
  • Security Isolation: Containers run inside a dedicated user-defined bridge network, ensuring database queries remain internal and not exposed to the public web.
  • AWS SDK Integration: The backend uses Boto3 to query S3 standard buckets, EC2 instance statuses, and CloudWatch metrics, falling back gracefully to mock configurations when AWS keys are absent.

07. Docker Container Architecture

Docker Compose coordinates the microservice stack. Isolated bridge networks guarantee database safety.

Compose Services

  • nginx: Serving static files directly and proxying HTTP requests to backend.
  • backend: Running Flask REST API and Socket.IO for real-time alerts.
  • mysql: MySQL 8 database storing system states. Uses Docker volumes for state persistence.
  • adminer: Database administration panel restricted to admin access.

09. Security Architecture

Authentication and authorization are managed through JWT security tokens, role structures, and AWS IAM read-only access roles.

Enterprise Guardrails

  • JWT Authentication: Secure token exchange model. Tokens expire in 480 minutes (configured in DB application settings), requiring refresh handshake protocols.
  • Role-Based Access Control: Strict separation of concerns (Administrator, Developer, Viewer roles).
  • AWS Access: Access to cloud dashboards uses a dedicated read-only IAM user (`sentinelops`) that only has the exact policies needed: `AmazonEC2ReadOnlyAccess`, `AmazonRDSReadOnlyAccess`, `AmazonS3ReadOnlyAccess`, and `CloudWatchReadOnlyAccess`.

14. Scalability & Future Expansion

Planned architectural upgrades will scale SentinelOps for multi-region high availability requirements.

Phase 1

Application Load Balancer (ALB) Setup

Deploy dual-zone EC2 instances behind an AWS Application Load Balancer to achieve high availability.

Phase 2

AWS Relational Database (RDS) Migration

Migrate the internal container-based MySQL database to Amazon RDS Multi-AZ configuration to secure automatic failover operations.

Phase 3

EKS Kubernetes Cluster Orchestration

Transition Docker Compose configurations to an Amazon EKS cluster to orchestrate service scaling dynamically based on traffic parameters.
